## 📝 Sobre o Projeto

O **AraMatch** é uma plataforma conceitual voltada para a conexão entre estudantes universitários por meio de afinidades sociais, pessoais e acadêmicas. O sistema tem como objetivo facilitar a criação de novas amizades e relacionamentos amorosos dentro do ambiente universitário, permitindo que os usuários encontrem pessoas com interesses semelhantes.

A proposta surgiu a partir da percepção de que muitos estudantes têm dificuldade em ampliar seu círculo social durante a vida acadêmica, especialmente quando desejam conhecer pessoas fora de sua turma, curso ou grupo de convivência. O AraMatch busca atender essa necessidade por meio de um sistema de sugestões, curtidas, matches e mensagens.

Neste repositório, o foco não está na codificação da aplicação, mas sim na **documentação técnica do projeto de software**. Foram produzidos os principais artefatos de análise e projeto, incluindo requisitos, regras de negócio, casos de uso, histórias de usuário, contratos de operação, arquitetura, diagramas UML e modelo de dados relacional.

## ✨ Funcionalidades Principais

As principais funcionalidades previstas para o sistema AraMatch são:

- 🔐 **Cadastro de usuário:** permite que estudantes universitários criem uma conta na plataforma.
- 🔑 **Login:** autenticação do usuário por e-mail e senha.
- 👤 **Edição de perfil:** permite alterar foto, biografia, curso, período, universidade e interesses.
- ⚙️ **Definição de preferências:** permite escolher o tipo de conexão desejada, como amizade, relacionamento amoroso ou ambos.
- 🎯 **Visualização de sugestões:** exibe perfis compatíveis com base nas preferências e interesses informados.
- ❤️ **Curtida de perfil:** permite que o usuário demonstre interesse por outro perfil.
- 💞 **Geração de match:** cria uma conexão quando dois usuários demonstram interesse mútuo.
- 💬 **Envio de mensagens:** libera conversa apenas entre usuários que possuem match ativo.
- 🚫 **Bloqueio de usuário:** impede novas interações com usuários indesejados.
- ⚠️ **Denúncia de usuário:** permite registrar denúncias sobre perfis ou comportamentos inadequados.
- 🛡️ **Gerenciamento de denúncias:** permite que o administrador analise denúncias e tome decisões de moderação.

## 🏗 Arquitetura

O AraMatch foi modelado com base em uma arquitetura em camadas, separando responsabilidades entre interface, aplicação, domínio e persistência de dados.

### Camadas previstas

| Camada | Responsabilidade |
|---|---|
| **Interface** | Responsável pelas telas utilizadas pelo usuário universitário e pelo administrador. |
| **Aplicação** | Responsável por controlar os fluxos do sistema e direcionar as ações para os serviços corretos. |
| **Domínio** | Contém as principais regras de negócio, como curtidas, matches, mensagens, denúncias e bloqueios. |
| **Dados** | Responsável pela persistência das informações no banco de dados. |

Essa arquitetura favorece a organização do sistema, a separação de responsabilidades, a manutenção futura e a rastreabilidade entre requisitos, modelos e dados.

## 🗄 Modelo de Dados

O modelo de dados do AraMatch foi elaborado com base nas classes do domínio e nos requisitos funcionais do sistema. A estrutura proposta utiliza banco de dados relacional, no qual as entidades principais são representadas por tabelas.

### Principais tabelas

| Tabela | Descrição |
|---|---|
| **USUARIO** | Armazena dados de identificação, autenticação e status dos usuários cadastrados. |
| **PERFIL** | Armazena informações públicas do usuário, como bio, curso, período, universidade e foto. |
| **PREFERENCIA** | Armazena filtros e preferências definidos pelo usuário. |
| **INTERESSE** | Armazena o catálogo de interesses disponíveis. |
| **PERFIL_INTERESSE** | Representa o relacionamento muitos-para-muitos entre perfil e interesse. |
| **CURTIDA** | Registra interações de interesse entre usuários. |
| **MATCH** | Registra conexões criadas por interesse mútuo. |
| **CONVERSA** | Representa o canal de comunicação associado a um match. |
| **MENSAGEM** | Armazena mensagens enviadas em uma conversa. |
| **DENUNCIA** | Armazena denúncias feitas por usuários. |
| **BLOQUEIO** | Registra bloqueios realizados por usuários. |

## 🚀 Instalação e Execução

> [!IMPORTANT]
> Este projeto não possui implementação executável nesta etapa. O repositório contém a documentação e os artefatos de modelagem do sistema AraMatch.

Para visualizar a documentação:

```bash
git clone <URL_DO_REPOSITORIO>
cd AraMatch
```

Em seguida, acesse a pasta:

```bash
cd docs
```

Abra o arquivo:

```bash
AraMatch-Documentacao-de-Projeto.pdf
```

Para editar ou visualizar os diagramas PlantUML, acesse:

```bash
cd plantuml
```

E abra os arquivos `.puml` em uma extensão compatível com PlantUML ou em um editor online.
